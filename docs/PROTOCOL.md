# Games Vault Protocol

Current architecture: `gv-web` gateway, `gv-server` host runtime, browser player. `gv-worker` is no longer a separate production process.

## Trust boundaries

| Actor | Auth mechanism | Notes |
|---|---|---|
| Browser → gv-web | Auth.js session + CSRF on mutations | Email/password users stored in Postgres |
| gv-server → gv-web | Bearer API key from pairing | API key is generated by `gv-server pair` and stored in config |
| Browser player → session | `worker_token`, `host_token`, `peer_token`, `room_token` | Short-lived/session-scoped tokens; never committed or logged as config |

## Pairing flow

```text
Browser/admin           gv-web                  gv-server host
     │                    │                           │
     │ generate code      │                           │
     ├───────────────────▶│                           │
     │ ABCD-EFGH          │                           │
     │                    │                           │
     │                    │   gv-server pair CODE     │
     │                    │◀──────────────────────────┤
     │                    │   claim + ROM roots       │
     │                    │──────────────────────────▶│
     │                    │   server_id + api_key     │
```

`gv-server pair <CODE> --gv-web-url <URL>` writes:

```toml
[gv_web]
url = "https://your-gateway.example"

[auth]
api_key = "gvsk_..."
server_id = "..."

[rom]
roots = ["/path/to/roms"]
```

## Host polling flow

```text
gv-server                         gv-web
   │                                 │
   │ GET/POST verify                 │
   ├────────────────────────────────▶│
   │                                 │
   │ poll for pending commands       │
   ├────────────────────────────────▶│
   │ commands + lease tokens         │
   │◀────────────────────────────────┤
   │                                 │
   │ execute command in-process      │
   │ notify/result                   │
   ├────────────────────────────────▶│
```

Commands include:

| Command | Purpose |
|---|---|
| `start_game` | Resolve ROM/core, create session, build WebRTC PC, optionally answer host SDP |
| `sdp_offer` | Exchange SDP for reconnects/guests |
| `stop_game` | End session |
| `browse_files` | Browse configured ROM roots |
| `scan_paths` | Scan ROM roots and report game files |

## Browser play flow

```text
Browser                    gv-web                    gv-server
   │                         │                           │
   │ POST start_game + SDP   │                           │
   ├────────────────────────▶│                           │
   │                         │ leased command            │
   │                         │◀──────────────────────────┤
   │                         │                           │
   │                         │ SDP answer / ready notify │
   │                         │◀──────────────────────────┤
   │ SDP answer              │                           │
   │◀────────────────────────┤                           │
   │                                                     │
   │ WebRTC media/DataChannel                            │
   └────────────────────────────────────────────────────▶│
```

For host reconnects and guest joins, gv-web issues/validates session-scoped tokens:

| Token | Purpose |
|---|---|
| `worker_token` | Browser polling proof for a queued command/session |
| `host_token` | Host authority for privileged DataChannel commands |
| `room_token` | Shareable guest join token |
| `peer_token` | Per-peer auth token for WebRTC/DataChannel auth |

## ICE/TURN config

Browsers fetch ICE config from:

```http
GET /api/ice-config
```

Response:

```json
{
  "iceServers": [
    { "urls": "stun:stun.l.google.com:19302" },
    {
      "urls": "turn:turn.example.com:3478",
      "username": "turn-user",
      "credential": "turn-password"
    }
  ],
  "iceTransportPolicy": "all"
}
```

Configure with:

| Env var | Description |
|---|---|
| `GV_ICE_STUN_URLS` | Comma-separated STUN URLs |
| `GV_ICE_TURN_URLS` | Comma-separated TURN URLs |
| `GV_ICE_TURN_USERNAME` | TURN username |
| `GV_ICE_TURN_CREDENTIAL` | TURN credential |
| `GV_ICE_TRANSPORT_POLICY` | `all` or `relay` |

No deployment domain or TURN credential should be hardcoded in client assets.

## Runtime notes

- `gv-server` owns emulator/core execution in-process.
- Libretro cores are resolved by platform and downloaded/cached when missing.
- GStreamer encodes H.264 video and Opus audio for WebRTC.
- Save SRAM/state paths are derived from ROM identity and configured save dirs.
- `GV_WORKER_HOST`/`GV_WORKER_PORT` are legacy env names for the browser-facing local player endpoint; they do not refer to a separate worker binary.
