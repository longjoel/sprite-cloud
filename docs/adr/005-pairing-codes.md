# ADR 005: 8-letter pairing codes

**Status:** Accepted  
**Date:** 2026-06-14

## Context

A user needs to pair their gv-server with their gv-web account. The pairing
flow requires a shared secret that the user can transfer from the browser
to the server.

## Decision

8-letter case-insensitive alphanumeric codes (Plex-style), 5-minute TTL,
one-time use. Generated by gv-web, displayed on the `/dev` dashboard,
typed into `gv-server pair`.

## Rationale

- **Easy to type**: 8 characters, case-insensitive, no symbols. A user
  can type `ABCD-EFGH` in seconds.
- **Hard to guess**: 36^8 ≈ 2.8 trillion combinations. Brute force is
  infeasible in a 5-minute window.
- **Plex precedent**: Users are familiar with this pattern from Plex
  media server pairing.
- **Self-expiring**: 5-minute TTL prevents stale codes from accumulating.
- **One-time use**: Claimed codes are deleted — no replay attacks.

## Consequences

- Codes are stored in memory on gv-web, not in the database (they expire
  quickly).
- If a code expires before the user types it, they generate a new one
  from the dashboard.
- Case-insensitive matching reduces the keyspace slightly (36 vs 62
  characters) but 36^8 is still sufficient for a 5-minute window.
