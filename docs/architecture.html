Pairing Flow

  • • gv-server starts, generates 4-char code
  • • Opens WS to gv-web, presents code
  • • User signs in, enters code on site
  • • gv-web confirms pair over WS
  • • Same code system for clients

gv-web (Cloud)

  • • Next.js website + signaling WS
  • • OAuth (GitHub, Google)
  • • Pairing registry
  • • PostgreSQL

gv-server (Home PC)

  • • Generates code, WS to gv-web
  • • Listens for play signals
  • • Spawns gv-worker per game
  • • Manages ROM library
