Games Vault Architecture

Polling · 8-char codes (5m TTL) · gv-server orchestrates · Worker is thin

Pairing (8 letters, 5m TTL)

  • • gv-server starts, generates code
  • • Polls gv-web w/ code
  • • User signs in, enters code
  • • Poll response returns auth_token
  • • All subsequent polls use token

gv-server (orchestrator)

  • • Polls GET /api/commands (2s)
  • • Spawns & manages workers
  • • Proxies SDP: web ↔ worker
  • • POST SDP answers to gv-web

gv-worker (thin)

  • • Load libretro core
  • • Read ROM, stream video
  • • Accept WebRTC SDP from server
  • • Return SDP answer to server
  • • Minimal code surface
